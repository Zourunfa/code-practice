# 哲学思维分析问题

## Overview

使用系统化的哲学框架分析问题，避免凭直觉或单一视角思考。通过多维度哲学分析获得深层洞察。

## When to Use

- 用户请求用哲学思维分析问题
- 面临人生/职业重大决策
- 需要从多角度审视复杂问题
- 遇到价值观或伦理困境

## 分析流程

### 1. 理解问题
清晰复述用户的问题，确保理解准确

### 2. 选择框架
根据问题性质选择 2-3 个最合适的哲学框架：

| 问题类型 | 推荐框架 |
|---------|---------|
| 存在/身份问题 | 存在主义、现象学 |
| 知识/认识问题 | 认识论、怀疑主义 |
| 实践/决策问题 | 实用主义、伦理分析 |
| 冲突/矛盾问题 | 辩证法、批判性分析 |
| 价值观/道德问题 | 伦理分析、存在主义 |

### 3. 系统分析
按以下结构输出：

```
## 问题理解
[复述问题]

## 哲学分析
### [框架1名称]
- 分析要点
- 洞察

### [框架2名称]
- 分析要点
- 洞察

## 综合洞察
[汇总各框架的洞见]

## 进一步思考
[基于分析提出 2-3 个开放性问题]
```

## 核心哲学框架

### 1. 本体论分析 (Ontological)
追问本质：什么是真正的"存在"？核心要素是什么？

### 2. 认识论分析 (Epistemological)
追问知识：我们知道什么？不知道什么？局限在哪？

### 3. 辩证法 (Dialectical)
正反合：正题(现状) → 反题(矛盾) → 合题(超越)

### 4. 实用主义 (Pragmatic)
效果导向：什么真正有效？后果是什么？

### 5. 存在主义 (Existentialist)
自由责任：我的选择是什么？承担什么责任？

### 6. 伦理分析 (Ethical)
- 功利主义：最大福祉
- 义务论：必须做什么
- 德性伦理：应成为什么样的人

### 7. 现象学 (Phenomenological)
悬置假设，回归现象本身

### 8. 批判性 (Critical)
质疑前提、权力关系、隐藏假设

## Common Mistakes

- 只用单一框架 → 选择 2-3 个互补框架
- 直接给结论 → 引导用户自己思考
- 过于学术化 → 保持可理解性
- 没有后续问题 → 提供开放性反思问题
