## 第二部分：核心技术亮点（15分钟）

### Slide 3: 即时通信系统 ⭐⭐⭐⭐⭐
**技术方案**：
- 腾讯云IM + MQTT双协议
- 观察者模式消息分发
- 离线消息缓存机制

**技术指标**：
- 消息送达率：99.9%
- 支持17种消息类型
- 断线重连时间：<3s

**代码示例**：
```typescript
// 观察者模式实现消息解耦
export abstract class TIMListener {
  abstract onMessageReceived(message: IChatMessage): void;
  abstract onSessionEnd(message: IChatMessage): void;
}
```

### Slide 4: 视频通话系统
**技术方案**：
- 腾讯云TRTC实时音视频
- 推拉流架构
- 信令通道（IM自定义消息）

**关键功能**：
- 视频/语音问诊
- 自动挂断（3分钟）
- 通话录制追踪

**状态机设计**：
```
NO_CALL → WAITING_CALL → KEEP_CALL → CALLING
```

### Slide 5: 灰度发布系统 ⭐⭐⭐⭐⭐
**三级灰度策略**：
1. 门店级灰度
2. 用户级灰度
3. 全局灰度开关

**技术实现**：
```typescript
@computed get isGray() {
  return this.isUserGray || this.isStoreGray || this.isGrayGlobal;
}
```

**业务价值**：
- ✅ 风险可控（问题影响范围可控）
- ✅ 平滑迁移（用户无感知升级）
- ✅ A/B测试（新旧版本对比）

### Slide 6: 风控系统 ⭐⭐⭐⭐
**多层次风控**：
```
统筹管控 → 前置审查 → 风控校验 → 重复用药 → 网院风控
```

**风控类型**：
- 合理用药检测
- 重复用药检测
- 长处方检测
- 医保资格校验

**拦截机制**：
- 拦截类：needStop=1，阻断流程
- 提示类：needStop=0，用户确认
- 静默类：不弹窗直接处理

### Slide 7: 全链路追踪系统 ⭐⭐⭐⭐
**processId设计**：
```typescript
// 格式：门店编码 + YYYYMMDDHHmmss
processId = `${storeCode}${dayjs().format('YYYYMMDDHHmmss')}`
```

**埋点体系**：
- 神策埋点：业务流程追踪
- 阿里云SLS：IM消息日志
- 自定义埋点：关键节点监控

**价值**：
- 问题定位时间缩短80%
- 完整还原用户操作路径

### Slide 8: 状态管理架构
**MobX响应式设计**：
```
AppStore (全局)
├── TIMStore (IM通信)
├── MediaStore (音视频)
├── GrayStore (灰度发布)
└── TrackStore (埋点追踪)
```

**核心特性**：
- @computed 计算属性自动缓存
- reaction 响应式副作用
- when 条件执行

## 第三部分：技术难点攻克（5分钟）

### Slide 9: 难点1 - IM消息可靠性
**问题**：
- 小程序后台运行时消息丢失
- 网络波动导致消息延迟

**解决方案**：
- 离线消息缓存（messageRecordList）
- 页面激活时重新分发
- 消息重发机制

**效果**：
- 消息送达率：92% → 99.9%

### Slide 10: 难点2 - 视频通话稳定性
**问题**：
- 网络波动导致卡顿、断连
- 资源泄漏导致内存溢出

**解决方案**：
- 自动重连机制
- 降级策略（视频→语音）
- 资源及时释放

**效果**：
- 通话成功率：85% → 95%

### Slide 11: 难点3 - 多场景业务复杂度
**问题**：
- 9种业务场景，代码耦合严重
- 新增场景开发周期长

**解决方案**：
- 场景识别 + 策略模式
- 配置化驱动
- 组件复用

**效果**：
- 新增场景开发：3天 → 1天
